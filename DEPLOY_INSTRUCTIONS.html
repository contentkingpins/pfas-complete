Deployment Instructions for PFAS Claim Website

Issue

The build is failing with the following error:

You're importing a component that needs `useEffect`. This React hook only works in a client component. To fix, mark the file (or its parent) with the `"use client"` directive.

Changes Made

We've made the following changes to fix the issue:

  1. Added "use client" directive to src/app/layout.tsx
  2. Moved metadata to a separate file: src/app/metadata.ts
  3. Created client components for functionality that requires client-side hooks:
    • ViewportHeightFix.tsx: Handles the 100vh issue on mobile browsers
    • MobileNav.tsx: Handles the mobile navigation menu with its state
    • Footer.tsx (CopyrightYear): Handles the dynamic year in the footer
  4. Updated amplify-build.sh to ensure it works correctly

How to Deploy

Follow these steps to deploy the changes:

  1. Open a terminal or command prompt
  2. Navigate to the project directory: 
    cd "C:\Users\asieg\pfas completed\pfas-claim-website"
  3. Add all changes to Git: 
    git add .
  4. Commit the changes: 
    git commit -m "Fix client component issues in layout.tsx"
  5. Push the changes to GitHub: 
    git push

Note: If you're having issues with Git commands, you can also use the GitHub Desktop application or the GitHub web interface to commit and push the changes.

Alternative: Manual Deployment

If you prefer to deploy manually:

  1. Log in to the AWS Management Console
  2. Navigate to AWS Amplify
  3. Select your PFAS claim website application
  4. Click on the "Redeploy this version" button or "Start a new build"

Verification

After deployment, verify that the website is working correctly by:

  1. Visiting the deployed URL
  2. Testing the mobile navigation
  3. Checking that the viewport height fix is working on mobile devices
  4. Verifying that the copyright year in the footer is displayed correctly
